1
Create Maven Web Project (Local)
🏗️ ▾

Create a new Maven web project and push it to GitHub from your local machine.

Create Maven project
$ mvn archetype:generate \ -DgroupId=com.yourname.app \ -DartifactId=mywebapp \ -DarchetypeArtifactId=maven-archetype-webapp \ -DinteractiveMode=false
Push to GitHub
$ cd mywebapp $ git init $ git add . $ git commit -m "initial commit" $ git remote add origin https://github.com/yourusername/mywebapp.git $ git push -u origin main
2
Launch EC2 Instance on AWS
☁️ ▾

Set up your AWS EC2 Ubuntu instance and configure the Security Group.

ℹ️ Launch EC2 → Ubuntu 22.04 LTS → t2.micro (free tier). Create or use existing key pair (.pem file).
Security Group Rules — Add Inbound Rules
# Rule 1: SSH Type: SSH | Port: 22 | Source: Anywhere IPv4 # Rule 2: All Traffic (for app access) Type: All traffic | Port: All | Source: Anywhere IPv4 # Rule 3: Custom TCP (Tomcat port) Type: Custom TCP | Port: 8080 | Source: Anywhere IPv4
⚠️ For production, restrict inbound rules to specific IPs. "Anywhere IPv4" is for learning/testing only.
Connect via MobaXterm / SSH
$ ssh -i your-key.pem ubuntu@<EC2-PUBLIC-IP>
3
Install Java & Maven on EC2
⚙️ ▾

Update the server and install Java (OpenJDK 17) and Apache Maven.

Update & Install Java
$ sudo su - $ apt update # update machine packages $ apt install openjdk-17-jre-headless -y # install java
Download & Install Maven
$ wget https://dlcdn.apache.org/maven/maven-3/3.9.14/binaries/apache-maven-3.9.14-bin.tar.gz # Extract the archive $ tar -zxvf apache-maven-3.9.14-bin.tar.gz # Remove the archive file $ rm -rf apache-maven-3.9.14-bin.tar.gz # Rename and move to /root/maven $ mv apache-maven-3.9.14 maven
Set Maven in PATH (only once)
$ echo "export PATH=/root/maven/bin:$PATH" >> ~/.bashrc $ source ~/.bashrc # Verify Maven installation $ mvn --version
4
Install & Configure Apache Tomcat 9
🐱 ▾

Download Tomcat 9, extract it, and configure user credentials for the Manager App.

Download & Extract Tomcat
$ wget (curl address) https://dlcdn.apache.org/tomcat/tomcat-9/v9.0.115/bin/apache-tomcat-9.0.115.tar.gz $ tar -zxvf apache-tomcat-9.0.115.tar.gz $ rm -rf apache-tomcat-9.0.115.tar.gz
Configure Tomcat Users
$ vi apache-tomcat-9.0.115/conf/tomcat-users.xml
<!-- Insert before </tomcat-users> --> <role rolename="manager-gui"/> <role rolename="manager-script"/> <role rolename="manager-jmx"/> <role rolename="manager-status"/> <user username="admin" password="admin" roles="manager-gui,manager-script,manager-jmx,manager-status"/> <!-- Save: Esc then :wq -->
Enable Remote Access to Manager App
$ vi apache-tomcat-9.0.115/webapps/manager/META-INF/context.xml
<!-- Comment out the Valve (add <!-- before and --> after) --> <!-- <Valve className="org.apache.catalina.valves.RemoteAddrValve" allow="127\.\d+\.\d+\.\d+|::1|0:0:0:0:0:0:0:1" /> --> <!-- Save: Esc then :wq -->
ℹ️ Commenting out the RemoteAddrValve allows access to the Manager App from any IP, not just localhost.
5
Clone Project & Build WAR File
🐙 ▾

Clone your GitHub repo on the EC2 instance and build the WAR file with Maven.

Clone from GitHub
$ git clone https://github.com/yourusername/mywebapp.git $ cd mywebapp
Build the project (generates .war)
$ mvn clean package # or: mvn clean install # WAR file will be at: target/mywebapp.war
6
Deploy WAR to Tomcat & Launch
🚀 ▾

Copy the WAR to Tomcat's webapps directory, start the server, and access your app.

Copy WAR file to Tomcat webapps
$ cp target/mywebapp.war apache-tomcat-9.0.115/webapps/ # ls apache-tomcat-9.0.115/webapps/ to verify
Start Tomcat Server
$ sh apache-tomcat-9.0.115/bin/startup.sh
Access the App in Browser
# Copy EC2 IPv4 Public IP from AWS console http://<EC2-PUBLIC-IP>:8080 # Manager App (login: admin / admin) http://<EC2-PUBLIC-IP>:8080/manager # Your deployed webapp http://<EC2-PUBLIC-IP>:8080/mywebapp
✅ Your web app is now live! Open the Manager App, login with admin/admin, and you'll see your deployed application listed.
Stop Tomcat Server (when needed)
$ sh apache-tomcat-9.0.115/bin/shutdown.sh
